01 / O que é Claude Code?

Claude Code — Visão Geral

Claude Code é a CLI oficial da Anthropic que integra o modelo Claude diretamente no seu terminal e IDE, permitindo desenvolvimento assistido por IA com acesso total ao seu projeto.

  • Terminal-first: roda direto no seu shell favorito
  • Contexto completo: lê, edita e cria arquivos no projeto
  • Extensível: Skills, Hooks, Agents e MCP servers
  • IDE integration: VSCode

Setup

Instalação, CLAUDE.md e configuração do ambiente

02 / Setup e Configuração

Começando com Claude Code

Instalação — 3 opções

  • npm: npm install -g @anthropic-ai/claude-code
  • Site: claude.ai/download (app desktop)
  • VSCode: extensão "Claude Code" no Marketplace

Requer Node.js 18+ e uma conta Anthropic (ou API key).

CLAUDE.md

Arquivo na raiz do projeto que define instruções persistentes: regras de encoding, arquitetura em camadas, skills e agentes disponíveis.

É o "contrato" entre você e a IA.

Permissões

Controle o que o Claude pode executar sem pedir aprovação:

"allow": [
  "Bash(python:*)",
  "Bash(bash:*)",
  "Bash(svn:*)",
  "Bash(curl:*)",
  "Skill(rodar-teste:*)"
]

Estrutura

.claude/
├── settings.json       # config global
├── settings.local.json # config local
├── skills/             # conhecimento sob demanda
├── agents/             # sub-agentes especializados
└── plans/              # planos de implementação

03 / Skills

Skills — Conhecimento Sob Demanda

Skills são arquivos .md em .claude/skills/ que ensinam o Claude a executar tarefas específicas do seu projeto.
  • Auto-trigger: carregam automaticamente quando o Claude detecta um contexto relevante
  • User-invocable: você chama manualmente via /nome-da-skill
  • Composição: uma skill pode invocar outras skills
  • Projeto-específica: cada projeto tem suas próprias skills

Skills do nosso projeto

  • /encoding — regras de ISO-8859-1
  • /sql-patterns — padrões de TDLQuery
  • /memory-management — Create/Free
  • /nomenclatura — prefixos de arquivos e classes
  • /componentes — TDL* vs padrão Delphi
  • /rodar-teste — testes DUnit
  • /revisar-bug — review automático
  • /jira-assistant — integração Jira

↓ pressione ↓ para ver como funciona

03 / Skills — Estrutura

Anatomia de uma Skill

Cada skill é um arquivo .md com frontmatter YAML + corpo Markdown:

---
name: encoding
description: Regras de encoding ISO-8859-1.
  Carregar SEMPRE que for editar .pas/.dfm
user-invocable: true
---
## Regras de Encoding
- NUNCA usar UTF-8 em .pas/.dfm/.dpr
- SEMPRE usar patch_delphi.sh para editar
}

name

Identificador da skill. Usado no /nome-da-skill.

description

O Claude usa essa descrição para decidir quando carregar automaticamente. Quanto mais específica, melhor.

user-invocable

Se true, aparece como comando /skill que você chama manualmente.

Corpo (Markdown)

Instruções que o Claude segue quando a skill é carregada: regras, exemplos e templates.

03 / Skills — Auto-trigger

Como o Auto-trigger funciona

O Claude lê a description de todas as skills e decide automaticamente quais carregar com base no contexto da conversa.

Você faz isso... O Claude carrega...
Edita um arquivo .pas /encoding + /nomenclatura
Escreve uma query TDLQuery /sql-patterns
Usa Create / Free /memory-management
Pede para revisar um bug /revisar-bug + /jira-assistant
Digita /rodar-teste /rodar-teste (manual)
💡 Você não precisa lembrar de carregar nada — o Claude detecta o contexto e carrega as skills certas sozinho. Mas também pode chamar manualmente com /skill.
📌 Hooks também existem no Claude Code (comandos automáticos em eventos como PreToolUse e PostToolUse), mas ainda não usamos no projeto.

04 / Agents

O que são Agents?

Agents são arquivos .md em .claude/agents/ que definem sub-agentes especializados com papel, ferramentas e regras próprias.

  • Contexto isolado: cada agent roda em processo separado
  • Ferramentas específicas: define quais tools o agent pode usar
  • Composição: um agent pode chamar skills e outros agents
  • Modelo configurável: pode usar Opus, Sonnet ou Haiku

Nosso Pipeline

Criamos um pipeline completo de desenvolvimento com 4 agents:

  • 01_AnalistaRequisitos — analisa o chamado
  • 02_Implementador — escreve o código
  • 03_TestadorUnitario — cria os testes
  • RevisorFisco — revisa o código

04 / Agents — Detalhes

Nossos 4 Agents

01 — Analista de Requisitos

  • Busca a issue no Jira via API REST
  • Baixa requisito do Google Docs (MCP)
  • Classifica: Bug, Task ou Epic
  • Gera PLANO_FISCO-XXXX.md estruturado

Modelo: Opus · Tools: Read, Glob, Grep, WebFetch, Bash, Skill

02 — Implementador

  • Lê o plano técnico gerado pelo Analista
  • Implementa código Delphi XE6 passo a passo
  • Pergunta units de referência antes de codificar
  • Usa patch_delphi.sh (encoding ISO-8859-1)

Modelo: Sonnet · Tools: Read, Write, Edit, Glob, Grep, Bash

03 — Testador Unitário

  • Analisa o código implementado
  • Cria testes DUnit com TDBTestCase
  • Usa builders de KrRepositorio.pas
  • Transação no SetUp, Rollback no TearDown
  • Usa a skill /rodar-teste para compilar e executar

Modelo: Sonnet · Tools: Read, Write, Edit, Glob, Grep, Bash

Revisor Fisco

  • Revisa código e patches SVN
  • Prioriza: Segurança → Memory → Performance
  • Detecta memory leaks, SQL injection, etc.
  • Gera relatório com severidade (Crítico/Alto/Médio/Baixo)

Modelo: Sonnet · Tools: Read, Grep, Glob (somente leitura)

05 / Demo — Revisor Fisco

Revisor Fisco ao Vivo

Vamos abrir o VSCode e chamar o Revisor Fisco para revisar código Delphi em tempo real.

O que ele faz

  • Analisa units .pas e patches SVN
  • Detecta memory leaks, SQL injection, performance
  • Classifica achados: Crítico → Alto → Médio → Baixo
  • Sugere correções com código
🔒 O Revisor tem acesso somente leitura (Read, Grep, Glob) — ele não modifica nada no projeto.

Truques e boas práticas para tirar o máximo do Claude Code

06 / Dicas e Truques

Boas Práticas

CLAUDE.md completo

Documente convenções e regras — o custo de contexto é muito menor que o custo de retrabalho.

Skills modulares

Uma skill por domínio, pequena e focada. Use triggers específicos para auto-carregamento.

Memory system

O Claude salva automaticamente suas preferências e correções. Você também pode pedir "lembre disso". Contexto persistido entre conversas.

Paralelismo

Lance múltiplos agents em paralelo para pesquisas independentes.

Review sempre

Revise diffs, rode testes e valide antes de commitar. Human-in-the-loop é essencial.

Prompts claros

Seja específico. Dê contexto, mostre exemplos e defina critérios de aceite.
